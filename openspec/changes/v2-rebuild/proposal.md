# Socratic Tutor v2.0 - 升级规划

## 当前系统问题汇总

### 架构层面
1. **状态管理混乱** - Zustand store 逻辑复杂，竞态条件多
2. **路由设计缺陷** - session ID 和 subject 耦合，切换逻辑混乱  
3. **前后端类型不匹配** - 多次出现 frontend/backend 类型不一致
4. **错误处理不足** - 很多 silent failures，用户得不到反馈

### 功能层面
1. **学科切换逻辑** - 核心功能有 bug，session 状态管理有问题
2. **UI/UX 不一致** - 中英文混合，dark mode 处理不当
3. **输入框问题** - 多次修复仍有瑕疵
4. **AI 对话流** - 没有真正的苏格拉底式追问，缺乏上下文记忆

### 技术债务
1. **Codex 外包遗留** - 多次迭代后代码风格不统一
2. **测试覆盖不足** - 关键流程缺少 E2E 测试
3. **Bundle 过大** - TipTap 导致 1MB+ bundle

---

## v2.0 目标

### 核心改进
1. **重新设计状态管理** - 简化 store，明确边界
2. **重构路由系统** - 清晰的 URL 设计，学科/会话分离
3. **强化类型系统** - 前后端共享 types，编译时检查
4. **完善错误处理** - 用户友好的错误提示

### 功能增强
1. **真正的苏格拉底对话** - 上下文感知，渐进式提问
2. **学科资料管理** - 上传教材、笔记、参考资料
3. **学习进度追踪** - 掌握度可视化，知识图谱
4. **多模态支持** - 图片、公式、图表

### 技术栈优化
1. **评估替代方案** - TipTap 是否真的需要？
2. **代码分割** - 按需加载，减小首屏
3. **测试策略** - E2E + 单元测试覆盖核心流程
