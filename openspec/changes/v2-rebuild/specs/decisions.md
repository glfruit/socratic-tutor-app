# Socratic Tutor v2.0 - 产品决策记录

## 用户决策 (2026-03-07)

### 1. 学习方式多样化 ✅
**决策**: 不只苏格拉底式，提供多种学习方式

**实现方案**:
- **苏格拉底式** (默认) - 提问引导
- **直接讲解式** - 适合快速获取知识
- **案例分析式** - 通过实例学习
- **对比辨析式** - 比较相似概念

**UI**: 学科选择后，询问用户偏好方式

### 2. 理解度评估策略 ✅
**决策**: 逐步升级
- 先简单规则（关键词匹配、回答长度）
- 规则不够用时再 AI 评估

### 3. 对话历史保存 ❓
**待决策**: 保存多少轮？

**建议**: 最近 10 轮（平衡成本效果）

**影响分析**:
- 全部保存: 数据库大，API 费用高
- 10 轮: 足够上下文，成本可控
- 智能摘要: 实现复杂，后续版本

### 4. 语音功能 ❌
**决策**: 移动版再考虑

## 学科学习 - 资料中心模式

### 核心流程
```
用户选择学科 → 询问是否有资料 → 
  ├─ 有资料: 上传资料 → 以资料为中心组织学习
  └─ 无资料: 进入通用知识库学习
```

### 资料运用方式（用户选择）
1. **系统阅读** - 按章节逐步学习
2. **问题导向** - 针对资料提出问题
3. **复习巩固** - 已有基础，快速过一遍
4. **查缺补漏** - 针对薄弱点学习

### 资料管理
- 支持 TXT/Markdown 上传
- 简单全文搜索（暂不需要向量检索）
- 学习进度与资料绑定

## 阅读功能 - 已决策 ✅

### 核心定位
**将 book-reader-notes 技能做成 Web 应用**

### 功能特点
1. **伴读模式** - 像有一个阅读伙伴，帮助理解探讨
2. **多种阅读模式** - 包括苏格拉底式、直接讲解等
3. **支持格式** - PDF + EPUB + TXT
4. **进度管理** - 页码/百分比 + 章节 + 时长 三种都包含
5. **笔记功能** - MVP 不做，后续版本添加

### 阅读 vs 学科学习
| 维度 | 学科学习 | 阅读 |
|------|----------|------|
| 目标 | 掌握系统知识 | 理解特定内容 |
| 内容 | 教材/课程 | 上传的书籍/资料 |
| AI角色 | 导师，引导学习 | 伴读，辅助理解 |
| 结构 | 固定大纲 | 书籍原有结构 |
| 进度 | 知识点掌握度 | 阅读完成度 |

### AI 伴读触发方式
- **主动**: 每章结束后自动提问
- **被动**: 用户选中文字后提问
- **混合**: 根据阅读行为（停顿、回翻）智能触发
