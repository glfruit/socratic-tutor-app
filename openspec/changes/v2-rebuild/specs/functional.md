# Functional Specification - Socratic Tutor v2.0

## 1. 系统概述

Socratic Tutor v2.0 是一个 AI 辅助学习平台，提供**学科学习**和**阅读伴读**两大核心功能。

### 1.1 核心定位
- **学科学习**: 系统化掌握知识，支持多种学习模式
- **阅读伴读**: 上传书籍/资料，AI 辅助理解和探讨

### 1.2 用户角色
- **学生**: 主要用户，学习和阅读
- **教师** (后续版本): 管理学生、查看进度

### 2.2 学科学习模块

#### 2.2.1 学科列表页
**URL**: `/learn`

**功能**:
- 显示所有学科（数学、物理、化学、生物、计算机科学）
- 每个学科显示名称、描述、知识点列表
- 水平选择器（小学/初中/高中/大学/研究生）

**需求**:
- 学科数据从后端 API 获取
- 水平选择影响 AI 提问深度
- 点击学科进入学习或询问是否有资料

#### 2.2.2 资料上传（可选）
**流程**:
```
用户点击学科 → 询问"是否有教材/资料？"
  ├─ 是 → 上传 TXT 文件 → 进入资料中心学习模式
  └─ 否 → 进入通用知识库学习模式
```

**资料运用方式**（用户选择）:
1. **系统阅读** - 按章节逐步学习
2. **问题导向** - 针对资料提出问题
3. **复习巩固** - 已有基础，快速过一遍
4. **查缺补漏** - 针对薄弱点学习

#### 2.2.3 学习方式选择
**用户选择学习方式**:
1. **苏格拉底式** (默认) - 提问引导，不直接给答案
2. **直接讲解式** - 清晰解释概念，适合快速学习
3. **案例分析式** - 通过实例学习
4. **对比辨析式** - 比较相似概念，深化理解

#### 2.2.4 对话界面
**URL**: `/learn/:subject/:sessionId?`

**布局**:
```
┌─────────────────────────────────────┐
│  物理  |  高中  |  苏格拉底式        │  ← 头部信息
├─────────────────────────────────────┤
│                                     │
│  🤖 AI: 你觉得力和运动有什么关系？  │  ← 消息区域
│                                     │
│  👤 你: 力是运动的原因？            │
│                                     │
│  🤖 AI: 这是一个常见的直觉...       │
│     你能举一个力作用但物体          │
│     不动的例子吗？                  │
│                                     │
├─────────────────────────────────────┤
│  [输入框...] [发送]                 │  ← 输入区域
└─────────────────────────────────────┘
```

**功能需求**:
- 支持 Markdown 渲染（粗体、列表等）
- 消息历史显示（保留最近 10 轮）
- 公式渲染（KaTeX）
- 发送消息后输入框清空
- 加载状态显示

#### 2.2.5 学习进度追踪
**维度**:
- 每个知识点的掌握度（BEGINNER → MASTERY）
- 学习时长统计
- 完成的知识点数

**展示**:
- 仪表盘：雷达图显示各学科掌握度
- 学习报告：周/月学习统计

### 2.3 阅读伴读模块

#### 2.3.1 书籍管理
**URL**: `/read`

**功能**:
- 书籍列表（封面、标题、作者、阅读进度）
- 上传新书（PDF/EPUB/TXT）
- 删除书籍

#### 2.3.2 阅读器
**URL**: `/read/:bookId/:chapterId?`

**布局**:
```
┌──────────┬───────────────────────┬──────────┐
│          │                       │          │
│  目录    │      阅读区域         │  AI 对话 │
│          │                       │   面板   │
│  第1章   │   这里是书籍正文...   │          │
│  第2章   │                       │  🤖 对   │
│  第3章   │   选中文字可提问      │  这段有  │
│          │                       │  什么疑  │
│          │                       │  问吗？  │
│          │                       │          │
└──────────┴───────────────────────┴──────────┘
```

**功能需求**:
- 目录导航（EPUB 有目录，PDF/TXT 按页/段落）
- 文字选中后可向 AI 提问
- AI 伴读模式选择:
  - 主动: 每章结束自动提问
  - 被动: 用户选中文本后提问
  - 混合: 根据阅读行为智能触发
- 阅读进度保存（页码/章节/百分比）

#### 2.3.3 AI 伴读 Prompt 设计
```
你是用户的阅读伙伴，正在一起阅读《书名》。

角色特点:
- 不直接给答案，通过提问帮助理解
- 可以解释生僻概念
- 引导联系实际例子
- 适时总结关键观点

当前阅读内容:
{context}

用户问题/选中内容:
{userInput}

回复风格:
- 简洁明了（不超过150字）
-  friendly 和 encouraging
- 使用中文
```

### 2.4 仪表盘与计划

#### 2.4.1 学习仪表盘
**URL**: `/dashboard`

**内容**:
- 总学习时长（今日/本周/本月）
- 各学科掌握度雷达图
- 近期学习活动时间线
- 待复习知识点提醒

#### 2.4.2 阅读仪表盘
- 阅读总时长
- 完成的书籍数
- 正在阅读的书籍进度
- 阅读速度统计（字/分钟）

#### 2.4.3 学习计划
- 设定学习目标（如：本周完成物理力学）
- 每日/每周计划
- 学习提醒

## 3. 技术要求

### 3.1 前端
- React + TypeScript + Vite
- TailwindCSS
- Zustand（单一 store，简化状态管理）
- react-markdown（渲染 Markdown）
- KaTeX（公式渲染）

### 3.2 后端
- Node.js + Express + TypeScript
- Prisma ORM
- PostgreSQL
- Redis（对话上下文缓存）

### 3.3 AI 集成
- DeepSeek API（主要）
- OpenAI（备选）

### 3.4 文件解析
- TXT: 原生支持
- EPUB: epub2 库
- PDF: pdf-parse 库

---

## 4. 非功能需求

### 4.1 性能
- 首屏加载 < 3s
- AI 响应 < 5s（流式输出）

### 4.2 兼容性
- 支持 Chrome, Firefox, Safari
- 响应式（桌面优先，平板适配）

### 4.3 安全
- JWT 认证
- API 限流
- 文件上传类型限制
